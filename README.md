# H_Tigran
## (Լաբորատոր աշխատանք 3)

Այս լաբորատոր աշխատանքի շրջանակներում ոչ միայն ուսումնասիրվել են լիցենզավորման սկզբունքները, այլև ամբողջությամբ կարգավորվել է կոմպիլյացիայի միջավայրը և զգալիորեն վերամշակվել է տրամադրված բազային կոդը՝ իրական անվտանգության և տրամաբանական պահանջներին համապատասխանեցնելու համար:

### 1. Աշխատանքային միջավայրի կարգավորում (Environment Setup)
* Քանի որ նախագիծը խիստ կապված է Windows API-ի և ապարատային ապահովման (WMI, iphlpapi) հետ, տեղադրվել է **Microsoft Visual C++ (MSVC) կոմպիլյատորը** (Visual Studio Build Tools 2022 փաթեթի միջոցով):
* Տեղադրվել և կարգավորվել է **CMake** գործիքը՝ System PATH-ում ավելացմամբ:
* Հետո նախագիծը հաջողությամբ հավաքվել է (build) `cmake ..` և `cmake --build .` հրամանների միջոցով:

### 2. Ֆայլում պահպանելու տրամաբանության արմատական ուղղում (Save Logic Fix)
* **Խնդիրը:** `main.cpp`-ում նախկինում գործող տրամաբանությունը (case 5) չէր պահպանում օգտագործողի ստեղծած իրական բանալին: Փոխարենը ստեղծվում էր կեղծ, մշտական (perpetual) լիցենզիա ("Test User" անունով) և գրանցվում ֆայլում:
* **Լուծումը:** `LicenseManager::saveToFile` մեթոդի ստորագրությունը փոխվել է `(const std::string& licenseKey, const std::string& filename)`: Այժմ ֆունկցիան փնտրում է մուտքագրված բանալին օպերատիվ հիշողության `licenses` վեկտորում և, միայն այն գտնելու դեպքում, ֆայլում պահպանում է ճշգրիտ տվյալները՝ մեքենայի ID-ն, տեսակը և ավարտի ժամկետը:

### 3. Ժամանակավոր լիցենզիայի ժամկետի պահպանում և վերականգնում (Expiration Date Handling)
* **Խնդիրը:** Ֆայլում չէր պահպանվում trial լիցենզիաների ավարտի ամսաթիվը, ինչի հետևանքով ծրագիրը վերագործարկելիս բոլոր լիցենզիաները դառնում էին անժամկետ:
* **Լուծումը:** * `LicenseKey` դասում ավելացվել է `getExpiryDate()` արժեքը վերադարձնող մեթոդ:
  * Ֆայլում պահպանելիս ավելացվել է նոր տող՝ `EXPIRY=...` (պահպանվում է Unix timestamp ձևաչափով):
  * `verifyFromFile` մեթոդը փոփոխվել է այնպես, որ կարդա `EXPIRY` դաշտը և այն համեմատի համակարգչի ընթացիկ ժամանակի հետ (`time(nullptr)`)՝ արգելափակելով մուտքը ժամկետանց լիցենզիաների դեպքում:

### 4. Նոր հնարավորություն՝ լիցենզիայի մնացորդային օրերի ստուգում (Check Remaining Days)
* Ավելացվել է նոր մեթոդ `LicenseManager::checkRemainingDays`:
* Մեթոդը որոնում է բանալին, ստուգում է արդյոք այն ժամանակավոր է, և եթե այո, ապա `difftime()` ֆունկցիայի միջոցով հաշվում է ավարտի և ընթացիկ ժամանակի միջև եղած տարբերությունը վայրկյաններով:
* Ստացված արժեքը բաժանվում է 86400-ի (օրվա վայրկյանների քանակը), և էկրանին արտածվում է ճշգրիտ մնացած օրերի քանակը: Այս ֆունկցիան կանչելու համար գլխավոր ընտրացանկում (menu) ավելացվել է 6-րդ կետը:
